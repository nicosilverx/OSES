The trigger statuses have been made to consistently handle race conditions on the update of Alarms and Schedule Tables.

In particular, in the multicore case we have a potential race condition due to the following\+:
\begin{DoxyItemize}
\item the handling of the trigger data structure is done in mutual exclusion using a spin lock.
\item the spinlock is released during the handling of the notifications. This is done because the time spent in handling notifications may be long (e.\+g., a Schedule Table activating a number of tasks at an expiry point.
\item when a core is handling the notifications at an expiry point related to an Alarm (Schedule Table), with the spin lock unlocked, a \mbox{\hyperlink{group__primitives-alarm_ga7bfac685beaa3a62872ed44161e07e09}{Cancel\+Alarm()}} (\mbox{\hyperlink{group__primitives-schtable_gad3d659c08b2702e817d42ed171b7dfd5}{Stop\+Schedule\+Table()}}) could be issued by another core on the same Alarm (Schedule Table). Once canceled (stopped), the Alarm (Schedule Table) could be restarted again.
\end{DoxyItemize}

Therefore, the status transitions of a trigger will be the following\+:
\begin{DoxyItemize}
\item O\+S\+E\+E\+\_\+\+T\+R\+I\+G\+G\+E\+R\+\_\+\+I\+N\+A\+C\+T\+I\+VE\+: Initial status, trigger inactive
\item O\+S\+E\+E\+\_\+\+T\+R\+I\+G\+G\+E\+R\+\_\+\+A\+C\+T\+I\+VE\+: Alarm or Schedule Table started
\item O\+S\+E\+E\+\_\+\+T\+R\+I\+G\+G\+E\+R\+\_\+\+E\+X\+P\+I\+R\+ED\+: The trigger fired. The status is changed to expired before leaving the critical section.
\item O\+S\+E\+E\+\_\+\+T\+R\+I\+G\+G\+E\+R\+\_\+\+C\+A\+N\+C\+E\+L\+ED\+: The status is set to canceled if while the trigger is expired and the spin lock released a \mbox{\hyperlink{group__primitives-alarm_ga7bfac685beaa3a62872ed44161e07e09}{Cancel\+Alarm()}} or \mbox{\hyperlink{group__primitives-schtable_gad3d659c08b2702e817d42ed171b7dfd5}{Stop\+Schedule\+Table()}} arrives from another core. At the end of the notification handling the status will be set back to O\+S\+E\+E\+\_\+\+T\+R\+I\+G\+G\+E\+R\+\_\+\+I\+N\+A\+C\+T\+I\+VE
\item O\+S\+E\+E\+\_\+\+T\+R\+I\+G\+G\+E\+R\+\_\+\+R\+E\+E\+N\+A\+B\+L\+ED\+: The status is set to reenabled if a canceled Alarm or Schedule\+Table is reenabled while a core is still handling the actions. At the end of the notification handling the status will be set to O\+S\+E\+E\+\_\+\+T\+R\+I\+G\+G\+E\+R\+\_\+\+A\+C\+T\+I\+VE with the new start time. 
\end{DoxyItemize}
