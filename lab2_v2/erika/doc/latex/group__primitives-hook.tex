\doxysubsection{Detailed Description}
On multicore systems, hooks are local callbacks to each core.

The \mbox{\hyperlink{group__primitives-hook_ga17050620b53ad1baf18088294604c919}{Error\+Hook()}} callback is called every time an error is detected inside an E\+R\+I\+K\+A3 primitive. The callback can be used to implement centralized error handling. A set of macros are also available to better understand the source of the error. In particular, \mbox{\hyperlink{group__primitives-hook_ga17050620b53ad1baf18088294604c919}{Error\+Hook()}} receives as parameter the error that is raised by the primitive. Then, a call to \mbox{\hyperlink{group__primitives-hook_gaaacd2429e582d3e58f45837b482a1936}{O\+S\+Error\+Get\+Service\+Id()}} returns informations about which primitive caused the error. Finally, calls to the macros O\+S\+Error\+\_\+\+X\+X\+X\+\_\+\+Y\+Y\+Y() returns the values of the {\itshape Y\+YY} parameter of the primitive {\itshape X\+XX}.

The \mbox{\hyperlink{group__primitives-hook_ga5cb66285b5a4c562ec9c47bd9f7eb2b6}{Startup\+Hook()}} callback is called inside the \mbox{\hyperlink{group__primitives-startup_ga5fb8a3c5837da7854c4da9972cefda96}{Start\+O\+S()}} primitive to implement the application startup procedure.

The \mbox{\hyperlink{group__primitives-hook_ga0aa20efaae2d6dfc08579cc92576a890}{Shutdown\+Hook()}} callback is called inside the \mbox{\hyperlink{group__primitives-startup_ga6df6f7c4e5965c72bd089c9f381f6ef2}{Shutdown\+O\+S()}} primitive to implement the Application specific shutdown procedures.

The \mbox{\hyperlink{group__primitives-hook_ga777f5858f7bea1dd2fc8b57710fdb6b9}{Pre\+Task\+Hook()}} and \mbox{\hyperlink{group__primitives-hook_ga163fbc3284a1b8fb48403ee11dc9587a}{Post\+Task\+Hook()}} callbacks are called respectively every time a task becomes the R\+U\+N\+N\+I\+NG Task, and every time a task stops to be the R\+U\+N\+N\+I\+NG Task.

The application can only call a limited set of primitives inside Hook functions.
